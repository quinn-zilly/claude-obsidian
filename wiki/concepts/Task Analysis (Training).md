# Task Analysis (Training)

The leg of [[Training Needs Analysis]] that produces a description of work activities — the **tasks** performed and the **KSAOs** (knowledge, skills, abilities, other characteristics) required to complete them. Undertaken **only after** organizational analysis confirms the company will invest, because it is time-consuming.

## Terms

- **Job** — a position requiring completion of certain tasks.
- **Task** — an employee work activity in a specific job.
- **KSAO** — Knowledge (facts/procedures), Skill (competency in a task), Ability (physical/mental capacity), Other (conditions: equipment, environment, time, safety, e.g., willingness to travel). See [[KSA Framework]].

## Four Steps

1. Select the job(s) to analyze.
2. Develop a preliminary task list (interview/observe expert employees + managers).
3. **Validate** the list — SMEs rate each task on **importance, frequency, difficulty** (sample questionnaire, Table 3.6).
4. Identify KSAOs per task; build a **task-KSAO grid** (Table 3.8; "X" marks required KSAOs).

## Selection Rule

Tasks that are **important, frequent, and moderate-to-high difficulty** → train. Unimportant or infrequent → don't. Important-but-infrequent or important-but-easy → judgment call, weighing error consequences. In the insurance-company territory-manager example, **importance** was the primary driver: "very important" tasks were designated for training regardless of frequency/difficulty.

## Vs Competency Models

Task analysis is **work/task-focused** ("what is accomplished"); [[Competency Modelling]] is **worker-focused** ("how it's accomplished"). Both feed course objectives and lesson plans.
