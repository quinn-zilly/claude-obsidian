# Transactive Memory Systems

## Definition

A **transactive memory system (TMS)** is a group-level cognitive structure in which knowledge is distributed across individual members, with shared awareness of *who knows what*. The group collectively "knows" more than any individual, and members can access each other's knowledge by knowing where to look.

Key components:
- **Specialization** — members hold different, complementary knowledge domains
- **Credibility** — members trust each other's expertise in their domains
- **Coordination** — members can locate and access each other's knowledge efficiently

## Why It Matters for Knowledge Integration

TMS is a primary micro-level mechanism for [[Knowledge Integration]]. Instead of requiring all knowledge to be in one place (or one person), TMS enables groups to integrate distributed specialist knowledge through coordinated access. This is why Grant (1996) argues the *firm* exists: to integrate specialist knowledge that individuals hold separately.

## TMS as a Knowledge Repository (Argote Framework)

In Argote & Miron-Spektor's (2011) framework, TMS is part of the **active context** — specifically embedded in the member-member network (social network) and the member-task network (who knows how to do what). TMS can be lost when:
- Key members leave the organization
- Team composition changes and tacit knowledge of "who knows what" dissipates
- Coordination patterns break down

## Implications for Organizational Learning

- Teams with well-developed TMS learn faster because they can efficiently access distributed expertise
- TMS is a form of organizational memory that is inherently social — it cannot be fully captured in documents or databases
- Building TMS requires time and shared work experience; it cannot be designed top-down
